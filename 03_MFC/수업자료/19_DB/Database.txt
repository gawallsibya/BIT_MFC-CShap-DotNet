1. 데이터베이스의 종류

   1) 계층형 데이터베이스(Hierarchical DBMS)
      데이터를 구성하는 레코드들이 계층구조로 구성됨
      데이터에 대한 접근은 하향식으로 처리됨
      DB 구조가 변경되면 응용 프로그램을 다시 개발해야 하는 단점

      DBMS(DataBase Management System)
      - 여러 개의 파일이나 응용프로그램에 의해 저장되고 처리되던 자료들을
         하나의 통합된 시스템으로 만들어 데이터의 가치를 더욱 높이는 기능을 함
        ex) Oracle, Access, MS_SQL

   2) 망형 데이터베이스(Network DBMS)
      DB의 구조가 지나치게 복잡해 질 수 있는 단점
      응용 프로그램 구현하기가 어려움

   3) 관계형 데이터베이스(Releational DBMS)
      기존의 DB가갖고 있던 데이터의 독립성 결여와 접근의 어려움을 개선
      오라클사에 의해 최초로 상용화됨
      데이터는 기본적으로 2차원의 테이블로 구성됨
      데이터간의 관계는 컬럼에 의해 정의됨



2. DBMS의 이점
   - 표준 파일 포멧으로 사용할 수 있다.
       DB를 통한 호환가능한 프로그램 개발 가능

   - 인덱스 파일 액세스가 용이
     키만 설정해 주면 빠른 레코드 검색이 가능

   - 데이터 무결성 보호를 갖는 프로시저를 설정 
     트랜잭션 처리(일련의 연관성 있는 변화들이 포함되며, 처리가 제대로 
     수행되지 않으면 기존 데이터가 보존됨)

   - 다중 사용자가 액세스 할 수 있도록 한다.
     레코드 Locking을 제공하여 액세스를 가능하도록 만들어 주고 또한 이 기능
     을 이용하여 데이터가 엉키는 것(복제 충돌)을 방지해 준다.


3. 데이터베이스의 주요 기능

   ODBC(Open DataBase Connectivity)
   - 출력 장치에 상관없이 TextOut()이라는 일관된 인터페이스로 문자열 출력
     서로 다른 DB를 동일한 방법으로 접근할 수 있게 인터페이스를 구성하는 것이
     ODBC임
     일반적으로 C/S 환경에서 리모트 DB를 핸들링할 때 사용됨
   - ODBC를 통해 Access, 오라클, SQL Server, FoxPro...등에 일관되게 접근

   DAO(Data Access Objects)
   - VB 3.0 을 발표하면서 MS에서JET 엔진 추가와 함께 발표함
     일반적으로 Access와 같은 독립 환경에서 로컬 DB를 핸들링할때 사용됨

   RDO(Remote Data Objects)
   - VB 4.0 발표와 함께 선보임 
     C/S 환경에서 DB를 보다 빠르고 쉽게 접근할 수 있게 함
   - SQL Server나 Oracle과 같은 중형 베이터 베이스를 접근하는데 적합한 방식임

   ADO(ActiveX Data Objects)
   - DAO와 RDO의 장점을 모아 만들어짐 OLE DB기반임
   - 적은 코드로 응용프로그램에 접근할 수 있게 함
   - ASP나 ADO를 사용할 수 있기 때문에 인터넷/인트라넷 환경에서 웹 페이지와 
     DB를 연결하여 동적인 웹사이트 구축시 사용됨
